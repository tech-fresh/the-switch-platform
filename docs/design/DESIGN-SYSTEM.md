# The Switch Platform — Design System

This file supports `docs/design/UI-UX-MASTER-GUIDE.md` and defines the MVP visual system.

## Design direction

The Switch Platform should feel:

- Premium.
- Vibrant.
- Minimal.
- Accessible.
- Educational.
- Fast.

Use the official Switch brand colours already present in the product and README/design references as the source of truth. Do not introduce new brand colours without approval.

## Colour rules

Use colour with purpose, not decoration.

Suggested balance:

- 70% neutral backgrounds.
- 20% Switch brand colour.
- 10% semantic accents.

Semantic use:

- Purple: brand, primary CTAs, active states, Power Grid.
- Blue: information, progress, guidance.
- Green: success and completion.
- Amber: warnings, upcoming exams, reminders.
- Red: errors only.
- Neutral: background, cards, body text.

## Accessibility

- Target WCAG 2.2 AA minimum.
- Never rely on colour alone.
- Pair colour with text, icon, or label.
- Keep visible focus states.
- Support high contrast mode.

## Typography

- One heading family.
- One body family.
- Clear hierarchy.
- Short paragraphs.
- Generous line height.
- No decorative fonts.

## Spacing

Use an 8-point spacing system:

- 8px: small gaps.
- 16px: related items.
- 24px: sections.
- 32–40px: major page blocks.

## Cards

Every card should have:

- One purpose.
- Consistent border radius.
- Consistent padding.
- Soft shadow or subtle border.
- One primary action at most.

## Buttons

Button types:

- Primary.
- Secondary.
- Outline.
- Ghost.
- Destructive only for high-risk actions.

Each screen should have one dominant button.

## Motion

Allowed:

- Fade.
- Slide.
- Hover lift.
- Progress fill.
- Skeleton loading.

Avoid distracting animation, flashing, excessive bounce, or motion that competes with learning.
