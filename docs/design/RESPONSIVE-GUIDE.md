# The Switch Platform — Responsive Guide

## Mobile first

Design mobile first, then scale up.

## Mobile

- One-column layout.
- Bottom navigation.
- Large touch targets.
- Sticky primary action where useful.
- No horizontal scrolling.

## Tablet

- Two-column layout where helpful.
- Keep primary action visible.
- Avoid dense dashboards.

## Desktop

- Use wider layouts for clarity, not clutter.
- Keep dashboard focused.
- Use whitespace generously.

## Breakpoint rule

Layouts must adapt without hiding core learning actions.

## Accessibility

Touch targets must be comfortable. Text must remain readable at adjusted sizes.
